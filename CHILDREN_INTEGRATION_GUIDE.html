1. What We Built

Before this integration, the newsletter picked a random children's product idea every week. There was no system, no guidance on what to actually build, and topics could repeat.

Now, every week automatically gets:

  1. A specific, pre-picked topic from a curated list of 52 (one per week for a full year)
  2. A one-page brief that tells you exactly what to build, how to build it in Canva, and how to price it
  3. The newsletter's Children's Activities section uses that same topic and links you back to the brief
  4. A no-repeat tracker that remembers every topic used so nothing comes up twice

2. How the No-Repeat System Works

There are 52 topics in a list inside the script — one for every week of the year. A simple JSON file (config/children_topics_log.json) acts as a memory log.

How It Tracks Topics
Alphabet Tracing Workbook
Week 1 · Jan 06
Number Tracing Workbook
Week 2 · Jan 13
Ocean Explorer Busy Book
Week 3 · Jan 20
Preschool Activity Binder
← next up
Color and Shape Flashcards
... 47 more topics waiting ...

Every time the script runs, it:

  1. Opens the log file and reads which topics have been used
  2. Picks the first one not in the list
  3. Adds it to the log with today's date
  4. Generates the brief for that topic
After 52 Weeks
The log resets automatically and the cycle starts over from Topic 1. You get a full year of unique topics with zero manual tracking.
How to Check What's Been Used
Open config/children_topics_log.json in any text editor. The history array shows every topic, week number, and date used.

3. Your Weekly Workflow

The only change to your existing routine is running one extra command at the beginning. Everything else stays the same.

Old Pipeline (what you ran before)

New Pipeline (run this instead — copy and paste the whole thing)

# NEW: generate children's brief first (picks this week's topic) python tools/children_brief_generator.py # then everything as before python tools/research_products.py && \ python tools/scrape_rss.py && \ python tools/scrape_reddit.py && \ python tools/draft_newsletter.py && \ python tools/format_email.py && \ python tools/send_email.py --dry-run
After reviewing the dry run
Run python tools/send_email.py && python tools/log_to_sheets.py to send for real and log the edition.

After the Pipeline Runs

  1. Open .tmp/unified_product_brief.html in your browser — this is your week's brief
  2. The brief tells you exactly what to build, step by step, in Canva
  3. Your newsletter (in .tmp/draft.md) already has the Children's section built around the same topic

4. Your Files — What Each One Does

Three files were added to your Newsletter folder. You don't need to edit any of them — just know they exist.

File What It Does Do You Edit It?
tools/children_brief_generator.py The main script. Picks this week's topic, generates the brief, saves everything. No. Just run it.
config/children_topics_log.json The memory. Keeps track of every topic used so far. Gets updated automatically every run. No. The script writes to it.
.tmp/unified_product_brief.html This week's brief. Opens in your browser. Gets overwritten each week with the new topic. No. Open it, don't edit it.
.tmp/children_brief_data.json A small data file that passes the topic from the brief generator to draft_newsletter.py. No. Internal use only.

And one existing file was updated:

File What Changed
tools/draft_newsletter.py Now reads .tmp/children_brief_data.json before calling Groq. Passes this week's topic into the prompt so the newsletter's Children's section matches your brief. Also adds a "📋 brief ready" note in that section.

7. Resetting or Skipping a Topic

Occasionally you might want to skip a topic or reset the whole list. Here's how:

?

Skip a topic you don't want this week

Open config/children_topics_log.json in a text editor. In the used_topics array, manually add the topic name you want to skip (copy it exactly as it appears in the script). The script will skip it and pick the next one.

↺

Start the whole list over from the beginning

Delete the contents of config/children_topics_log.json and replace with:

{"used_topics": [], "current_week": 0, "history": []}

The next run will start again from Topic 1: Alphabet Tracing Workbook.

+

Add your own topics to the list

Open tools/children_brief_generator.py. Find the ALL_TOPICS list. Add a new entry in this format:

{"topic": "Your Topic Name", "age": "3–6", "format": "PDF printable"},

8. FAQ

Why do I run children_brief_generator.py BEFORE research_products.py?
The brief generator saves this week's topic to .tmp/children_brief_data.json. Then when draft_newsletter.py runs, it reads that file and tells Groq which specific children's topic to use. If you run them the other way, the newsletter won't have the topic yet.
What happens if I forget to run children_brief_generator.py?
The newsletter still runs fine — the children's section just won't be locked to a specific topic. You'll see the message: No children's topic found — run children_brief_generator.py first for a specific topic.
Will it use the same topic twice if I run the script twice in the same week?
No — the log is updated on the first run. Running it again immediately would advance to the NEXT topic. So only run children_brief_generator.py once per week, at the start of the pipeline.
The brief is a local file — can I share it or link to it in the email?
The brief is just for you — it's your reference while building in Canva. The newsletter email itself already contains all the key information (product, build prompt, pricing tips) in the Children's Activities section. Your subscribers get everything they need in the email. You get the detailed brief locally.
How do I see all the topics in advance?
Open tools/children_brief_generator.py and look at the ALL_TOPICS list. All 52 topics are there in order. Topic 1 is Alphabet Tracing Workbook, Topic 52 is Multiplication Flashcards 1–5.
What if the brief generation fails because of the Groq API?
The topic is already saved to the log before the Groq call. So the topic won't repeat. Just re-run python tools/children_brief_generator.py to regenerate the brief. The log won't advance — it already has this week's topic recorded.
